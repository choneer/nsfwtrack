# 当前目标：Phase 3-B4 — 媒体清理恢复中心

当前稳定版与最新 Release 为 `v1.0.6`，Phase 3-B1 / B2 已正式发布。
Phase 3-B3 / B4 仅记录在 CHANGELOG Unreleased，版本保持 1.0.6，Schema 保持 2。

## 目标

隔离 B3 内部安全锚点，并提供只读恢复状态查看入口。

## 任务

- 新增登录保护的媒体清理恢复中心
- 识别 `.cleanup-anchor-*` 安全锚点和 `recovered-*` 恢复文件
- 展示路径、大小、SHA-256、合法性及封面 / 头像引用
- 区分已引用锚点、未引用锚点、损坏锚点和正常恢复文件
- 将安全锚点从普通媒体库、重复组、上传去重及 A3/A4 候选中排除
- `recovered-*` 继续作为正常媒体，并增加恢复文件筛选和标识
- 在数据健康页增加锚点残留与引用状态审计
- 支持路径和 SHA 搜索、稳定排序、20 条分页
- 同步中英文、CHANGELOG Unreleased、文档和测试

## 边界

- 本阶段只读，不删除、移动、改名或自动恢复文件
- 不自动解除或迁移任何数据库引用
- 不把文件名前缀之外的普通媒体误判为内部锚点
- 不改变 B1/B2 分组规则和 B3 整理流程
- 不请求网络，不使用 AI 或图像识别
- 不修改版本 1.0.6、Schema 2、迁移、依赖或 Docker/CI
- 不创建 tag / Release，不部署 N100

## 完成标准

- 安全锚点不再出现在普通媒体和候选流程
- 已引用安全锚点不再出现在普通媒体，但仍可通过恢复中心和数据健康页定位
- 恢复文件保持可用并可筛选
- GET 全程零写入
- 普通媒体、B1/B2、A3/A4 行为无回归
- pytest、pip check、Docker 和 Actions 通过
