## 二、本轮任务 1：合集数据模型

请新增合集 / 清单相关模型。

建议模型：

```text
Collection
ItemCollection
```

建议字段：

### Collection

```text
id
name
description
created_at
updated_at
```

要求：

1. `name` 必填。
2. `name` 不允许为空字符串。
3. `description` 可选。
4. `created_at` / `updated_at` 使用现有项目风格。
5. 删除合集时，只删除合集和关联关系，不删除条目。
6. 合集名称建议唯一；如果实现唯一约束，需要有友好错误提示。
7. 不要把合集和标签混用。
8. 不要自动创建合集，除非用户明确在合集表单中创建。
9. 不要引入外部数据源。
10. 不要新增依赖。

### ItemCollection

```text
item_id
collection_id
created_at
```

要求：

1. 一个条目可以属于多个合集。
2. 一个合集可以包含多个条目。
3. 同一个条目不能重复加入同一个合集。
4. 删除条目时应清理关联关系。
5. 删除合集时应清理关联关系。
6. 不要删除被关联的条目。
7. 优先使用 SQLAlchemy ORM 关系。
8. 关联表应有合理的唯一约束，避免重复关联。

## 四、本轮任务 3：合集创建 / 编辑 / 删除

请新增合集创建、编辑、删除能力。

### 创建合集

要求：

1. 需要登录。
2. `name` 必填。
3. `name` 去除首尾空格。
4. 空名称应显示错误提示。
5. 重复名称应显示错误提示。
6. `description` 可选。
7. 创建成功后跳转到合集详情页或合集列表页。
8. 创建成功显示 flash success。
9. 创建失败显示 flash error。
10. 不自动添加任何条目。

### 编辑合集

要求：

1. 需要登录。
2. 可以修改名称。
3. 可以修改描述。
4. 名称仍然必填。
5. 重复名称应显示错误提示。
6. 编辑成功显示 flash success。
7. 编辑失败显示 flash error。
8. 不影响合集内条目关系。

### 删除合集

要求：

1. 需要登录。
2. 删除前有浏览器 confirm。
3. 页面有危险操作提示。
4. 删除合集只删除合集和关联关系。
5. 不删除条目。
6. 删除成功显示 flash success。
7. 删除失败显示 flash error。
8. 删除后返回合集列表页。
9. 不实现软删除 / 回收站。
