# 当前目标：Phase 3-D1 — Unreleased 集成总审查与开发冻结

## 目标

对 Phase 3-B3 至 C5 进行最终集成、安全和完成度审查，
确认所有计划功能能够作为一个完整版本进入发布准备状态。

## 任务

- 审查 B3-B6、C1-C5 的页面、服务、路由、模板和测试衔接
- 检查 Data Health 每种 finding 是否有正确入口、说明或安全拒绝
- 检查媒体库、重复组、恢复中心、跳过项和根目录诊断之间的导航
- 检查所有写操作的登录保护、确认、事务、竞态和失败报告
- 检查所有 GET 页面保持数据库和文件系统零写入
- 检查删除、恢复、引用迁移后 Data Health 与媒体库状态一致
- 检查备份导出、验证、恢复 dry-run 和实际恢复不受新增功能影响
- 搜索 TODO、FIXME、占位实现、失效入口、错误状态码和文档矛盾
- 修复审查中确认的真实问题，并补充最小必要测试
- 新增 Phase 3 完成度审计文档并同步现有状态文档

## 边界

- 不新增产品功能、数据模型或自动化操作
- 不扩大外部来源、网络请求、远程图片、爬虫、识别或 AI 范围
- 不修改版本 1.0.6、Schema 2、迁移或依赖
- 不修改 Docker/CI，除非发现并证明存在真实缺陷
- 不创建 tag、Release 或部署 N100
- 不为了测试数量重构正常代码
- 没有真实问题时不得制造修复提交或虚构缺陷

## 完成标准

- 所有 Unreleased 页面均可达，无死链接或孤立入口
- Data Health finding、操作入口及完成后的状态闭环一致
- 高风险操作在成功和所有失败路径下保持文件与数据库安全
- 备份、恢复、导入、升级和设置流程无回归
- 中英文键完整，无模板缺键或原始错误泄露
- 文档中的版本、Schema、提交、测试数量和发布范围一致
- 专项集成测试、全量 pytest、pip check、Docker 和 Actions 通过
- 输出明确结论：开发完成，或列出仍需处理的真实阻塞问题
