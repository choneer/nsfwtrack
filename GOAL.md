# GOAL.md

# NSFWTrack 当前开发目标

本轮开发目标：
在保持 Phase 1 本地 MVP 边界不变的前提下，完成 Phase 1 收尾体验修复和部署文档增强。

本轮重点是：

* 统一页面成功 / 失败提示
* 优化导入、备份、表单操作的用户反馈
* 整理 N100 / Docker 部署说明
* 做一次 Phase 1 整体自查
* 评估并尽量处理测试 warning

不新增外部源，不新增复杂业务功能。

---

## 一、必须遵守的项目边界

NSFWTrack 是项目名称。

Phase 1 严格限定为：

> 本地单用户内容记录器 / 收藏管理器 MVP。

本轮只允许实现：

* 页面提示体验优化
* 成功 / 错误提示统一
* 表单操作反馈优化
* 导入 / 备份 / 恢复页面提示优化
* README 部署文档增强
* REVIEW / TASKS 文档同步
* 测试 warning 评估与低风险修复
* Phase 1 整体 checklist 自查

本轮禁止实现：

* 外部内容源
* 爬虫
* adapter
* 远程图片拉取
* cookie / token 管理，登录 session 和语言偏好 session 除外
* 自动同步
* 多源搜索
* 随机探索接口
* 推荐系统
* AI 助手
* URL 导入备份
* 云端备份
* 定时备份
* 覆盖式恢复
* 复杂权限系统
* 多用户系统

如果开发中发现上述超范围内容，不要实现，直接跳过并说明原因。

---

## 二、本轮任务 1：统一提示系统

当前各页面的成功、失败、错误提示需要统一。

请实现或整理一个轻量提示机制。

要求：

1. 支持成功提示。
2. 支持错误提示。
3. 支持普通信息提示。
4. 提示内容支持中文 / English。
5. 不引入大型前端框架。
6. 继续保持 FastAPI + Jinja2 + HTMX。
7. 提示样式尽量统一。
8. 不要输出完整异常堆栈到用户页面。
9. 不要在提示中输出密码、密钥、cookie、token 等敏感信息。

可选实现方式：

* 使用 session flash message
* 或使用 query 参数传递简短状态
* 或在现有页面上下文中传入 message / error

优先选择简单、清晰、可维护的方案。

## 三、本轮任务 2：优化现有页面反馈

请重点优化以下页面或操作的提示体验：

1. 登录成功 / 登录失败。
2. 退出登录。
3. 新增条目成功 / 失败。
4. 编辑条目成功 / 失败。
5. 删除条目成功 / 失败。
6. 标签 / 创作者操作成功 / 失败。
7. CSV / JSON 导入成功 / 失败。
8. JSON 备份预览成功 / 失败。
9. JSON 备份恢复成功 / 失败。
10. 备份文件过大、格式错误、schema 错误时的提示。

要求：

* 用户能明确知道操作结果。
* 中文 / English 都需要支持。
* 不要因为提示优化引入复杂前端逻辑。
* 不要改变现有 API 字段名。
* 不要破坏现有测试。
