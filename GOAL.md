# GOAL.md

# NSFWTrack 当前开发目标

本轮开发目标：  
进入 Phase 2-B2，在保持项目本地单用户边界不变的前提下，增强统计面板，让用户能更清楚地了解本地数据概况、状态分布、评分分布、标签排行、创作者排行和最近活动情况。

本轮重点是：

- 统计面板结构增强
- 总览统计卡片
- 状态分布统计
- 评分分布统计
- 标签使用排行
- 创作者关联排行
- 最近创建 / 最近更新趋势
- 空数据状态提示
- 纯 HTML / CSS 轻量可视化
- 中英文文案覆盖
- 相关测试与文档更新

本轮不新增外部内容源，不做推荐系统，不做 AI 分析，不引入图表库。

- 本地统计查询

- 前端构建流程

## 二、本轮任务 1：统计数据服务整理

请整理统计相关后端逻辑。

建议新增或整理 service：

```text
app/services/stats.py
```

如果当前已有统计逻辑，可以在现有结构中整理，但不要让 router 变得过长。

要求：

1. 统计数据只来自本地 SQLite。
2. 不请求外部网络。
3. 不读取外部数据源。
4. 不做推荐或预测。
5. 不做 AI 分析。
6. 不新增数据库字段。
7. 不修改数据库结构。
8. 不新增依赖。
9. 优先使用 SQLAlchemy ORM 或清晰、安全的查询方式。
10. 统计结果结构清晰，方便模板渲染。
11. 空数据时返回稳定结构，不要 500。
12. 异常输入或无数据时页面正常显示。
