# GOAL.md

# NSFWTrack 当前开发目标

本轮开发目标：  
进入 Phase 2-B1，在保持项目本地单用户边界不变的前提下，对现有页面做移动端 / 响应式 UI 打磨，让 NSFWTrack 在手机、平板和桌面端都更容易使用。

本轮重点是：

- 全站响应式布局优化
- 顶部导航 / 语言切换 / 登录状态移动端适配
- 条目列表页移动端优化
- 详情页移动端优化
- 批量编辑区域移动端优化
- 导入页面移动端优化
- 表单页面移动端优化
- 备份 / 统计页面基础响应式优化
- 保持桌面端体验不退化
- 中英文文案不破坏
- 相关测试与文档更新

本轮不新增业务功能，不进入 Phase 3，不接入外部内容源。

## 二、本轮任务 1：全站响应式基础

请检查并优化全站基础布局。

要求：

1. 页面在桌面端保持现有可用体验。
2. 页面在手机宽度下不应出现明显横向滚动。
3. 主内容区域在窄屏下应有合理边距。
4. 表格、按钮组、筛选区、表单区在窄屏下不应挤压错位。
5. 长文本、标签、创作者名称应能正常换行。
6. 页面不应因为长标题或长标签撑破布局。
7. 操作按钮在移动端应更容易点击。
8. 不要引入新的 UI 框架。
9. 优先使用现有 CSS 文件或模板内已有样式结构。
10. 如需要新增 CSS，请保持简单、可维护。

建议重点检查：

- `app/templates/base.html`
- `app/templates/items.html`
- `app/templates/_item_card.html`
- `app/templates/detail.html`
- `app/templates/item_form.html`
- `app/templates/import.html`
- 备份页面模板
- 统计页面模板

## 三、本轮任务 2：顶部导航移动端适配

请优化顶部导航。

要求：

1. 桌面端导航保持清晰。
2. 移动端导航不应挤成一行。
3. 语言切换入口应在移动端可用。
4. 登录 / 登出入口应在移动端可用。
5. 主要页面入口应在移动端可见或可换行显示。
6. 项目名 NSFWTrack 不要翻译。
7. 不要实现复杂侧边栏。
8. 不要引入汉堡菜单框架。
9. 可以使用简单换行、flex-wrap、分组布局解决。
10. 不要破坏现有登录流程。

```bash

curl -s -o /dev/null -w "%{http_code}\n" "http://localhost:8000/login"
```
