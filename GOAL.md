# GOAL.md

# NSFWTrack 当前开发目标

本轮开发目标：  
进入 Phase 2-A4，在保持项目本地单用户边界不变的前提下，增强导入功能，让用户更容易把本地数据批量导入 NSFWTrack。

本轮重点是：

- CSV 导入模板下载
- JSON 导入模板下载
- 导入字段说明
- CSV 字段映射
- 导入错误行提示
- 导入预览增强
- 导入结果摘要
- 中英文文案覆盖
- 相关测试

本轮不接入外部内容源，不做爬虫，不做推荐系统，不做 AI 助手。

## 二、本轮任务 1：CSV 导入模板下载

请新增 CSV 导入模板下载功能。

要求：

1. 提供一个下载 CSV 模板的入口。
2. 可以放在现有导入页面。
3. 模板文件应包含表头。
4. 模板文件应包含 1 到 2 行示例数据。
5. 模板内容只用于本地导入说明，不包含任何外部数据源。
6. 下载接口必须要求登录。
7. 文件名建议：

```text
nsfwtrack-import-template.csv
```

建议 CSV 字段：

```text
title,summary,status,rating,note,tags,creators,extra
```

字段说明：

- `title`：必填，条目标题
- `summary`：可选，条目描述
- `status`：可选，内部状态值
- `rating`：可选，1 到 5
- `note`：可选，短评 / 备注
- `tags`：可选，多个标签用分号分隔
- `creators`：可选，多个创作者用分号分隔
- `extra`：可选，JSON 字符串

注意：

- 不要翻译 CSV 表头字段名。
- 页面展示说明需要中文 / English。
- CSV 文件内容可以使用英文表头。
- 示例数据不要包含外部 URL 或真实敏感信息。

```json
        "source": "manual"
      }
    }
  ]
}
```

要求：

- 不请求外部 URL。
- 不包含真实外部内容源。
- 不把 JSON 字段名翻译。
- 页面说明支持中文 / English。

## 五、本轮任务 4：CSV 字段映射

请为 CSV 导入增加简单字段映射能力。

目标：

> 当用户的 CSV 表头不完全等于 NSFWTrack 默认字段时，可以在预览阶段选择哪些列对应哪些字段。

第一版只需要简单实现。

要求：

1. 用户上传 CSV 后，预览页面应读取 CSV 表头。
2. 如果 CSV 表头与默认字段一致，可以自动匹配。
3. 如果 CSV 表头不一致，用户可以手动选择映射。
4. 支持映射到以下目标字段：
   - title
   - summary
   - status
   - rating
   - note
   - tags
   - creators
   - extra
   - ignore / 忽略该列

5. `title` 映射必须存在，否则不能确认导入。
6. 重复映射到同一个目标字段时，应安全提示错误。
7. 没有被映射的列应忽略。
8. 不要做复杂转换表达式。
9. 不要做外部字段自动识别服务。
10. 不要引入新的依赖。
11. 映射结果只用于本次导入，不需要保存为长期配置。

实现方式可以简单：

- 上传 CSV → 显示预览和字段映射表单
- 用户确认映射 → 执行导入

如果当前导入流程已经是“上传 → 预览 → 确认”，请在预览阶段加入字段映射。
