# 当前目标：Phase 4-M3 — 增量媒体索引与扫描中心

## 目标

为本地媒体建立可重建的派生索引，
避免每次打开媒体页面都重新读取并哈希全部文件，
同时保留现有文件系统安全验证和写操作边界。

## 功能范围

### 1. Schema 3 媒体索引

新增派生索引表，至少记录：

- media_path，唯一
- basename、parent_directory、extension、mime_type
- size、sha256
- valid / damaged 状态
- recovered 状态
- device、inode、modified_ns、changed_ns
- directory mapping token
- first_seen_at、last_seen_at、indexed_at

新增索引状态表，至少记录：

- 最近一次增量扫描时间
- 最近一次完整验证时间
- 最近扫描结果与错误
- 当前索引条目数
- 当前媒体根身份
- 索引格式版本

索引是派生缓存，不是业务真相：

- 不进入备份导出
- 不从备份恢复
- 恢复数据库后自动标记失效
- 可以随时完整删除并重建
- 索引损坏不得影响条目、创作者、引用或媒体文件

### 2. Schema 2 → 3 迁移

- 增加正式 MigrationStep
- dry-run 展示新表、索引和约束
- precheck / postcheck 完整
- 旧 Schema 2 数据原样保留
- 迁移只创建空索引，不在数据库事务内扫描媒体文件
- 新数据库直接以 Schema 3 建立
- 升级失败不得留下半完成版本记录

### 3. 安全增量扫描

增量扫描必须使用现有 FD / O_NOFOLLOW 安全遍历语义。

每次扫描：

- 遍历媒体根并取得当前路径和身份
- identity 与缓存完全一致时复用 SHA、MIME 和有效性
- 新增或身份变化时重新安全读取、校验格式并计算 SHA
- 已从文件系统消失的索引项移除
- symlink、特殊文件、unsupported、unreadable 和 entry_error 继续准确统计
- cleanup anchor 和 upload residue 保持现有隔离语义
- recovered 普通媒体正常索引
- 根目录或父目录映射变化时不得错误复用旧记录
- 扫描失败不得提交半套索引

单次索引刷新使用事务整体替换或等价原子提交，
读页面只能看到旧的完整快照或新的完整快照。

### 4. 完整验证与重建

增加两种操作：

#### 增量刷新

- 复用身份未变化的索引项
- 只重新读取新增或变化文件
- 报告 reused / new / changed / removed / rehashed / damaged / skipped

#### 完整验证

- 不信任任何缓存 SHA 或有效性
- 重新读取和校验所有普通媒体
- 重新生成完整索引
- 报告旧索引与新扫描的变化
- 可用于修复索引损坏或确认磁盘状态

两种操作都只修改派生索引，不修改媒体文件和业务引用。

### 5. 扫描中心

新增登录保护页面：

- 当前索引状态
- 最近增量扫描和完整验证时间
- 当前文件、有效、损坏、恢复、跳过数量
- 上次扫描耗时
- reused / rehashed / new / changed / removed 统计
- 最近一次变化明细
- 增量刷新按钮
- 完整验证 / 重建预览与确认
- 索引不可用时显示明确降级状态

GET 页面数据库与文件系统零写入。

### 6. 只读页面接入索引

以下页面优先使用当前完整索引快照：

- 媒体库
- 目录浏览
- 重复媒体列表和重复组
- 硬链接 alias 审计
- 媒体匹配与未匹配候选

要求：

- 排序、筛选、分页和统计结果与现有扫描语义一致
- 页面显示索引时间和是否可能过期
- 索引缺失、失效或 Schema 未升级时安全降级或提示刷新
- 不因索引缓存跳过登录保护

以下路径仍必须执行即时文件系统重验，不能只信任索引：

- 单文件详情中的安全事实
- rename / move
- 批量 rename / move
- alias keeper 归一化
- duplicate cleanup
- damaged cleanup
- recovery / anchor 操作
- 引用修复
- 所有会修改数据库引用或媒体文件的 POST

索引可以用于定位候选，但不能作为写操作授权依据。

## 安全边界

- 不增加后台线程、任务队列或定时扫描
- 不在页面 GET 中隐式写索引
- 不请求网络
- 不增加爬虫、远程图片、识别、推荐或 AI
- 不把索引当作文件存在、身份或内容正确的最终证据
- 不修改现有媒体文件
- 不修改条目、创作者、标签、集合或引用
- 不保存敏感凭据
- 应用版本保持 1.0.6
- Schema 从 2 升级到 3
- 不创建 tag、Release，不部署 N100

## 完成标准

- Schema 2 → 3 dry-run、升级和全新初始化全部通过
- 增量扫描只重新读取新增或变化文件
- 父目录替换、inode 复用、mtime/ctime/size 变化不会错误命中缓存
- 扫描异常时旧索引保持完整可用
- 完整验证可以从空表或损坏索引恢复
- 主要只读页面结果与完整扫描结果一致
- 所有写操作继续即时重验真实文件系统
- 备份不包含派生索引，恢复后索引正确失效
- 中英文、文档和测试完整
- pytest、pip check、Docker、迁移验收和 Actions 通过
