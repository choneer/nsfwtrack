## 二、本轮任务 1：高级筛选

请增强条目列表页的筛选能力。

至少支持以下筛选条件：

### 1. 关键词搜索

- 继续支持标题模糊搜索。
- 保留现有搜索能力，不要破坏旧功能。

### 2. 状态筛选

支持按状态筛选：

- 想看
- 在看
- 已看
- 喜欢
- 不喜欢
- 忽略

要求：

- 英文字段值继续使用现有内部枚举，不要翻译 API 字段名。
- 页面展示文案需要支持中文 / English。

### 3. 标签筛选

- 支持选择一个标签进行筛选。
- 第一版只做单标签筛选即可。
- 不要在本轮做复杂多标签 AND / OR 逻辑。

### 4. 创作者筛选

- 支持选择一个创作者进行筛选。
- 第一版只做单创作者筛选即可。

### 5. 评分筛选

- 支持按最低评分筛选。
- 例如：只看评分大于等于 3 / 4 / 5 的条目。
- 如果当前数据没有评分，应正常显示空结果，不要报错。

### 6. 时间筛选

支持按创建时间或更新时间筛选。

第一版只做简单范围：

- 最近 7 天
- 最近 30 天
- 最近 90 天
- 全部

不要做复杂日期选择器。

### 高级筛选通用要求

- 所有筛选都只针对本地 SQLite 数据。
- 不允许请求外部接口。
- 未登录访问仍应跳转登录页。
- 筛选参数应使用 query string，方便复制链接和刷新页面后保留状态。
- 无结果时需要有清晰空状态提示。
- 中文 / English 都要支持。

## 三、本轮任务 2：排序功能

请为条目列表页增加排序功能。

至少支持：

1. 最新创建优先
2. 最早创建优先
3. 最近更新优先
4. 最早更新优先
5. 标题 A-Z
6. 标题 Z-A
7. 评分从高到低
8. 评分从低到高

要求：

- 默认排序建议为“最近更新优先”。
- 排序参数使用 query string。
- 切换排序时应尽量保留当前筛选条件。
- 排序文案支持中文 / English。
- 不要改变数据库结构，除非确实发现当前字段不足。
- 不要引入全文搜索引擎或复杂搜索库。

---

## 四、本轮任务 3：分页增强

请增强列表分页体验。

要求：

1. 支持分页大小选择。
2. 可选分页大小建议：
   - 10
   - 20
   - 50
   - 100
3. 默认分页大小建议为 20。
4. 分页大小使用 query string，例如：
   - `?page_size=20`
5. 切换分页大小时保留当前筛选和排序条件。
6. 页码跳转时保留当前筛选、排序和分页大小。
7. 非法 `page` / `page_size` 参数应安全回退到默认值。
8. 不要因为非法参数导致 500 错误。

---

## 五、本轮任务 4：列表页 UI 体验增强

请优化条目列表页的使用体验。

要求：

1. 在列表页顶部增加筛选区域。
2. 筛选区域包含：
   - 关键词搜索框
   - 状态选择
   - 标签选择
   - 创作者选择
   - 最低评分选择
   - 时间范围选择
   - 排序选择
   - 分页大小选择
   - 搜索 / 应用筛选按钮
   - 清空筛选按钮

3. 展示当前筛选条件。
   - 例如：当前状态、当前标签、当前创作者、当前排序。
   - 没有筛选时显示“全部条目”。

4. 保留用户输入。
   - 用户搜索或筛选后，表单中应显示当前选择。
   - 刷新页面后状态不丢失。

5. 空结果提示。
   - 没有匹配条目时显示明确提示。
   - 提供“清空筛选”入口。

6. 继续保持轻量结构。
   - 不引入 React / Vue。
   - 继续使用 FastAPI + Jinja2 + HTMX。
   - 不做复杂前端状态管理。
