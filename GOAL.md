# GOAL.md

# 当前目标：Phase 2-E1 保存筛选视图 / 常用视图

请先读取 `RULE.md`，再执行本文件。  
`RULE.md` 是长期通用规则，本文档只描述本轮开发目标。  
如果 `RULE.md` 与本文档冲突，以 `RULE.md` 的安全边界为准。

---

## 一、目标

新增本地保存筛选视图能力。

用户可以把条目列表页当前常用筛选条件保存为一个命名视图，之后一键应用。

典型用途：

- 保存“未完成”
- 保存“高评分”
- 保存“某标签”
- 保存“某创作者”
- 保存“某合集”
- 保存“最近更新”
- 保存“待回顾”

本轮只做本地 saved views。

本轮不做：

- AI 推荐
- 自动分类
- 智能视图
- 外部内容源
- URL 导入
- 爬虫
- adapter
- 云同步
- 多用户共享视图
- 复杂权限
- 前端框架重构

---

## 二、数据库变更

本轮允许新增一个本地表：

- `saved_views`

建议字段：

- `id`
- `name`
- `query_string`
- `created_at`
- `updated_at`

要求：

- 不修改已有表字段
- 不删除已有表
- 不新增其他表
- 不新增依赖
- 使用现有 SQLite / SQLAlchemy 风格
- 兼容旧数据库启动，通过现有 `create_all` 机制创建新表
- 旧数据不受影响

---

## 三、保存内容

保存的是条目列表页筛选 / 排序参数。

允许保存的参数应来自现有列表页支持的参数，例如：

- 搜索关键词
- 状态
- 评分
- 标签
- 创作者
- 合集
- 排序字段
- 排序方向
- 每页数量

要求：

- 不保存登录信息
- 不保存 CSRF / session / cookie
- 不保存外部 URL
- 不保存未知危险参数
- 不保存分页页码，避免打开视图后跳到过期页
- query_string 应经过白名单过滤和稳定排序，便于比较和测试

---

## 四、页面与交互

在条目列表页新增：

- 保存当前视图入口
- 视图名称输入
- 已保存视图列表
- 应用视图入口
- 更新当前视图入口，可选
- 删除视图入口

建议新增或复用路由：

- `POST /saved-views`
- `POST /saved-views/{id}/update`
- `POST /saved-views/{id}/delete`
- `GET /saved-views/{id}/apply`

要求：

- 创建 / 更新 / 删除必须登录
- 创建 / 更新 / 删除必须 POST
- 应用视图可以 GET，但只能跳转到条目列表页，不修改数据
- 删除必须有 confirm
- 视图名称不能为空
- 视图名称过长要限制
- 视图名称重复时给出清楚提示
- 非法 id 不 500
- 移动端可用
- 不破坏现有筛选 / 排序 / 分页
- 不破坏语言切换
- 不破坏登录 / 登出

---

## 五、安全边界

必须遵守：

- 保存视图只是保存本地查询参数
- 不允许执行任意 SQL
- 不允许保存完整 URL 后直接跳转
- 不允许跳转到站外地址
- 不允许自动修改条目
- 不允许自动创建标签 / 创作者 / 合集
- 不允许推荐内容
- 不允许请求外部网络
